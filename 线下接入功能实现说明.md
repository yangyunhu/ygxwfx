# 线下接入数据源按钮功能实现说明

## 功能概述

在"无感数据自定义"页面的"数据线下导入维护"标签页中,当用户在"无感数据模式维护"中将某个数据源的接入模式改为"线下接入"后,该数据源会自动以按钮形式显示在顶部的sub-tabs区域,与"闸机门禁"和"食堂用餐记录"并列显示。

## 实现位置

### 1. DataCustom.vue - 主页面组件

**文件路径**: `src/views/DataCustom.vue`

#### 关键修改点:

1. **模板部分 (第24-49行)**
   - 在sub-tabs区域添加了动态循环,显示所有线下接入的数据源按钮
   ```vue
   <div
     v-for="source in offlineSources"
     :key="source.id"
     class="sub-tab sub-tab-button"
     :class="{ active: dataType === source.dataType }"
     @click="switchDataType(source.dataType)"
   >
     {{ source.sourceName }}
   </div>
   ```

2. **计算属性 - offlineSources (第608-627行)**
   - 从localStorage读取当前选中组织机构的配置
   - 过滤出`sourceType === 'offline'`的数据源
   - 转换为按钮所需的格式(包含id、sourceName、dataType)

3. **计算属性 - activeListData (第647-673行)**
   - 添加了对线下数据源的判断逻辑
   - 当`dataType`以`offline_`开头时,使用示例数据显示

4. **样式部分 (第1368-1390行)**
   - 添加了`.sub-tab-button`样式类
   - 支持hover和active状态
   - 与原有sub-tab样式保持一致

### 2. DataConfig.vue - 配置页面组件

**文件路径**: `src/views/DataConfig.vue`

#### 关键功能:

1. **添加数据源对话框**
   - 用户可以选择数据来源为"单位线下自行导入"
   - 保存后会将`sourceType`设置为`'offline'`

2. **数据存储**
   - 使用localStorage存储每个组织机构的配置
   - Key格式: `org_config_${orgId}`
   - Value: JSON数组,包含所有数据源配置

## 工作流程

### 步骤1: 配置线下接入数据源

1. 进入"无感数据配置"页面(DataConfig.vue)
2. 选择一个组织机构
3. 点击"添加数据源"
4. 选择数据源(如:南网商旅通)
5. **数据来源**选择"单位线下自行导入"
6. 设置同步频率和接入状态
7. 点击"确定"保存

### 步骤2: 查看按钮显示

1. 切换到"无感数据自定义"页面(DataCustom.vue)
2. 进入"数据线下导入维护"标签页
3. 在顶部sub-tabs区域会看到新增的按钮(如:"南网商旅通")
4. 按钮与"闸机门禁"、"食堂用餐记录"并列显示

### 步骤3: 使用线下数据源

1. 点击对应的数据源按钮(如:"南网商旅通")
2. 页面会切换到该数据源的数据列表
3. 可以进行查询、导入、导出等操作

## 技术细节

### 数据流转

```
DataConfig.vue (配置页面)
    ↓
用户选择"单位线下自行导入"
    ↓
保存到localStorage: org_config_${orgId}
    ↓
DataCustom.vue (自定义页面)
    ↓
offlineSources计算属性读取localStorage
    ↓
过滤出sourceType === 'offline'的数据源
    ↓
渲染为sub-tab按钮
    ↓
用户点击按钮
    ↓
switchDataType()切换dataType
    ↓
activeListData根据dataType显示对应数据
```

### localStorage数据结构

```javascript
// Key: org_config_YN_CSG_DEPT_001
// Value:
[
  {
    id: 1234567890,
    sourceCode: "nanwang_shanglv",
    sourceName: "南网商旅通",
    sourceType: "offline",  // 关键字段
    syncFreq: "每日同步",
    status: "sync"
  },
  {
    id: 1234567891,
    sourceCode: "canteen_system",
    sourceName: "食堂管理系统",
    sourceType: "online",
    syncFreq: "实时同步",
    status: "sync"
  }
]
```

### 按钮标识规则

- 固定数据源: `dataType = 'gate'` 或 `'canteen'`
- 线下数据源: `dataType = 'offline_' + source.id`
  - 例如: `offline_1234567890`

## 注意事项

1. **必须刷新页面**: 在DataConfig中修改配置后,需要刷新DataCustom页面才能看到新按钮
2. **组织机构关联**: 线下数据源按钮只在配置了该数据源的组织机构下显示
3. **数据展示**: 目前线下数据源使用gateData作为示例数据,实际应该从服务器或专门的存储中获取
4. **localStorage清理**: 如果localStorage被清空,配置会丢失,需要重新配置

## 后续优化建议

1. **数据持久化**: 为每个线下数据源创建独立的数据存储
2. **数据导入**: 完善线下数据的上传和解析功能
3. **权限控制**: 根据不同用户角色显示不同的数据源按钮
4. **缓存优化**: 避免频繁读取localStorage,可以添加缓存机制
5. **错误处理**: 添加更完善的错误提示和数据验证
6. **批量操作**: 支持批量配置多个数据源为线下接入

## 测试方法

1. 打开浏览器开发者工具(F12)
2. 进入Application/Storage → Local Storage
3. 查看`org_config_*`键的值
4. 确认`sourceType`字段为`"offline"`
5. 刷新页面查看按钮是否显示
